Make a targeted content change to the website, then commit and deploy.

The user will describe what they want changed. The argument passed to this command is: $ARGUMENTS

Steps:
1. Read `index.html` to understand the current content and find the right section
2. Make only the change the user described — don't touch anything else
3. After editing, read back the changed section to verify it looks correct
4. Run /ship to commit and push

Guidelines for edits:
- Change only the text/content described; preserve all HTML structure, classes, and attributes
- Keep the tone consistent with the rest of the page: formal, understated, professional
- Don't add marketing language or superlatives
- If the user's request is ambiguous (e.g. "update the about section" without saying what to), ask one clarifying question before editing
